# Changelog

## 1.0.18 — 03.03.2026

### Ядро / Monitoring
- События (Events): прекращён спам одинаковыми `DOWN` при долгом падении и ретраях — события пишутся только при смене состояния (UP/DOWN/DNS/ISSUE), как было раньше.
- UI (Detail): полоска статусов и фон графика теперь учитывают эскалацию `ISSUE → DOWN` по `issue_escalate_minutes` и рисуют красным, когда монитор уже в `DOWN`, даже если ошибка — timeout.
