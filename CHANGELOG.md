# Журнал изменений

## 1.0.10 — 19.02.2026

### Архитектура
- Единый проброс языка интерфейса в API-запросах через `Accept-Language`.
- Меню-вкладки теперь используются не только для UI, но и как дополнительный серверный guard (см. «Безопасность»).

### Безопасность
- Zero-trust по вкладкам: `menu_permissions` группы теперь **реально ограничивает доступ** к страницам/API даже при наличии прав у роли (403 вместо «можно зайти по ссылке»).
- Мониторинг: роль `analyst` получила только права просмотра (без доступа к управлению/настройкам).

### Вкладки / Навигация
- При попытке открыть недоступную вкладку (например `dashboard`) приложение автоматически переадресует на первую доступную вкладку вместо экрана `forbidden`.
- Настройки: вкладка «Основные» стала персональной (доступна при наличии `app.view`), чтобы пользователи могли менять язык/пароль/персональные опции.

### Аккаунты (группы/роли)
- Группы: исправлена ошибка `400 Bad Request` при создании/редактировании, когда список участников отправлялся в неправильном формате.
- Группы: «Участники» теперь инициализируются/обновляются без `F5` и корректно показываются в режиме редактирования.
- Роли: добавлен поиск по правам в модальном окне и увеличена ширина модалки.
- Роли: добавлены «пакеты прав по вкладкам» (быстрое добавление/удаление безопасного набора прав просмотра, без выдачи manage-настроек).
- Шаблоны ролей обновлены.

### UI/UX
- Аккаунты: кнопки действий для групп/ролей переведены на текстовый вид (редактировать/удалить/сохранить/отмена).
- Настройки: убран «шум» 403 в консоли — секции/запросы инициализируются только при наличии доступа.

### Локализация (i18n)
- Аккаунты: создание/смена пароля — локализованы ошибки политики пароля (RU/EN).
- Аккаунты: русифицированы сообщения валидации вида `role required`.
