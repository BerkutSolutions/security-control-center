# Журнал изменений

## 1.1.0 — 04.03.2026

### Математические модели (критическое изменение)
- Введена серверная поведенческая модель риска (zero-trust): анализ активности пользователя по окнам времени, расчет score по формуле, анти-false-positive демпферы.
- Реализован механизм `step-up` при срабатывании модели: обязательная проверка пароля + второй фактор (`TOTP` или `passkey`).
- Добавлена эскалация временной блокировки при повторных неуспешных `step-up` попытках: 1 минута -> 15 минут -> 1 час.
- Добавлена отдельная документация по формальной части модели и параметрам: `docs/ru/behavior_risk_model.md`.
- Для healthcheck добавлен отдельный контроль поведенческой модели с нейтральным статусом при выключении.
- Для модели мониторинга (скоринг инцидентов) добавлен отдельный контроль в healthcheck; модель должна быть включена постоянно.
- Скорректированы пороги и cold-start подавление поведенческой модели, чтобы step-up был достижим при реально аномальной нагрузке.

### UI
- Исправлено отображение статусов и причин блокировки учетных записей.
- Добавлены модальные окна для блокировки пользователя (время + причина) и для согласования экспорта.
- Убраны браузерные `alert/prompt` в ключевых пользовательских сценариях, заменены на внутренние уведомления.
- Добавлены глобальные toast-уведомления справа снизу (автоскрытие 5 секунд, ручное закрытие).
- Добавлен центр уведомлений (иконка колокольчика) с удалением одного/всех уведомлений.
- Добавлены индикаторы-счетчики в боковом меню для согласований, инцидентов, задач, мониторинга и документов в работе.
- Добавлены улучшения профиля пользователя (иконка, hover-карточка, перенос пользовательских настроек).
- Для редактора и просмотрщика документов добавлено отображение выбранных ACL (пользователи/роли) в том же стиле, что при создании документа.
- В боковом меню выровнены и опущены вниз кнопки профиля/выхода; версия приложения центрирована внизу колонки.
- Hover-карточка профиля ограничена границами окна приложения (без выхода за край).
- Улучшен формат ошибок в уведомлениях: для серверных ошибок показывается код HTTP и endpoint запроса.
- Для Home mode предупреждение о встроенном HTTPS скрывается при фактическом доступе по `https://` (в том числе за reverse-proxy).
- Для страницы «Настройки» верхние inline-плашки переведены в toast-уведомления (справа снизу), чтобы не дублировать сообщения вверху вкладки.
- Глобальный toast-модуль теперь при загрузке страницы подхватывает и скрывает оставшиеся inline-`alert` (без перезагрузки), чтобы в интерфейсе оставались только динамические уведомления.

### Ядро
- Исправлена логика согласования экспорта документов:
- согласующий должен иметь ранг не ниже инициатора;
- для дефолтного `admin` подтверждение может дать только `superadmin`;
- статусы экспортного согласования приведены к циклу `review -> approved/returned`.
- Восстановлено отображение согласований экспорта после фактического экспорта (история не теряется).
- Добавлена серверная модель поведенческого риска с расчетом аномалий по действиям пользователя.
- Реализован `step-up` сценарий: пароль + второй фактор (`TOTP` или `passkey`) при обнаружении аномалии.
- Добавлена эскалация блокировок при неуспешном `step-up`: 1 минута -> 15 минут -> 1 час.
- Исправлена ошибка `500` при сохранении runtime-настроек (`/api/settings/runtime`) при отсутствии аудит-хранилища: добавлена безопасная запись аудита с `nil`-проверкой.
- Исправлена кросс-БД несовместимость runtime-настроек для поля `behavior_model_enabled` (SQLite/PostgreSQL): чтение нормализовано через `CAST(... AS TEXT)`, запись переведена на bool-тип.

### Вкладки и уведомления
- Реализовано live-обновление счетчиков без перезагрузки страницы.
- Для задач исключены из подсчета статусы `Готово` и `Бэклог`.
- Добавлен учет черновиков, документов на согласовании и документов, отправленных на доработку, в индикаторах документов.
- Исправлена навигация по уведомлениям: переход ведет к целевой сущности (задача/инцидент/согласование).
- Исправлен счетчик колокольчика: теперь он строго соответствует фактическому списку уведомлений (без «фантомных» значений после удаления).
- Фоновый опрос уведомлений переведен в «тихий» режим (`X-Berkut-Background: 1`), чтобы он не создавал ложные события «просмотр списка» в журналах действий.

### Healthcheck (после логина)
- Добавлен отдельный пункт проверки zero-trust поведенческой модели.
- Если поведенческая модель выключена, показывается нейтральный серый статус (без красного крестика).
- Добавлен отдельный пункт для модели мониторинга (скоринг инцидентов): она должна быть включена постоянно; при выключении отображается ошибка проверки.
- Модель скоринга мониторинга принудительно зафиксирована во включенном состоянии на сервере (политика «всегда включено»).
- В проверках hardening добавлено определение эффективного HTTPS за доверенным reverse-proxy (по `X-Forwarded-Proto`), чтобы Home mode не давал ложное предупреждение при реальном TLS.

### Безопасность
- Усилена zero-trust проверка на сервере для сценариев согласования и экспорта.
- Для новых endpoint'ов модели риска добавлена обязательная проверка сессии и состояния пользователя.
- Добавлен аудит событий модели риска и step-up:
- `security.behavior.stepup_required`;
- `security.behavior.stepup_password_ok`;
- `security.behavior.stepup_failed`;
- `security.behavior.stepup_success`.
- Добавлены/обновлены миграции БД для хранения событий поведения и состояния риска пользователя.

### Инфраструктура и прокси
- Исправлен источник `502 Bad Gateway` в `nginx` после пересоздания контейнера приложения:
- включен динамический DNS-resolve через Docker resolver (`127.0.0.11`);
- `proxy_pass` переведен на переменные upstream-адреса, чтобы nginx обновлял IP сервиса `berkut`/`onlyoffice` без ручного вмешательства.

### Настройки
- В «Укреплении безопасности» добавлен чекбокс включения поведенческой модели аномалий.
- В baseline hardening добавлена проверка состояния поведенческой модели.
- В «Укреплении безопасности» добавлено окно «Анализ активности»: выбор пользователя, текущий риск-скор, признаки аномалии, вклад метрик и причины срабатывания.

### Локализация (i18n)
- Добавлены RU/EN ключи для:
- step-up процесса (статусы, ошибки, подсказки);
- переключателя поведенческой модели и ее статусов в hardening;
- сообщений экспортного согласования и уведомлений.
- Исправлены проблемные строки с кракозябрами и несовпадениями локализации.
- Добавлены RU/EN расшифровки audit-событий поведенческой модели в журнале действий (`settings.behavior_model.toggle`, `security.behavior.*`).

### Документация

- `docs/ru/behavior_risk_model.md`.
- Добавлена английская версия документа по модели поведенческого риска:
