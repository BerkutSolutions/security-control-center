# Журнал изменений

## 1.1.1 — 05.03.2026

### UI
- В таблицах приложения включен перенос длинных значений по словам, чтобы ссылки и длинные поля не выходили за пределы столбцов.
- Таблица сертификатов мониторинга приведена к унифицированному стилю строк/статусов (с бейджами статуса).
- В карточке «Истекание TLS сертификата» реализован рабочий список порогов уведомлений с чекбоксами включения.
- Кнопка + в настройках TLS-уведомлений теперь действительно добавляет порог и показывает подтверждение в интерфейсе.

### Ядро
- Поведение определения клиентского IP за reverse-proxy исправлено: добавлен безопасный fallback для контейнерного proxy-сценария при пустом списке trusted proxies.
- Добавлена поддержка нескольких порогов TLS-уведомлений (`tls_expiring_rules`) с обратной совместимостью для старого поля `tls_expiring_days`.
- Логика генерации событий/уведомлений TLS обновлена: срабатывание происходит при пересечении любого включенного порога (30/14/7 и пользовательские).

### Инфраструктура и прокси
- Для docker-развертывания задан дефолт `BERKUT_SECURITY_TRUSTED_PROXIES=172.16.0.0/12` (можно переопределить через `.env`).

### Безопасность
- Сохранен strict CSP (`style-src 'self'`); внешние inline-инъекции браузерных расширений продолжают блокироваться политикой безопасности.
