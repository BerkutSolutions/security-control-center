# Security Policy

## Supported versions

Security fixes are provided for the latest release branch (`latest`).

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Send a private report with:
- affected version and deployment mode;
- reproduction steps or proof of concept;
- impact assessment.

Contact: `ogsecs@gmail.com`

If you cannot use email, open a minimal public issue without exploit details and request a private channel.

## Response targets

- Initial triage response: up to 3 business days.
- Status update after triage: up to 7 business days.
- Fix timeline depends on severity and release risk.
