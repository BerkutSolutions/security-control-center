# Compatibility Policy

Политика определяет правила совместимости между релизами и действия оператора.

## Поддерживаемые переходы

1. Обязательно поддерживается переход `latest-1 -> latest`.
2. Для переходов через несколько минорных версий требуется промежуточное обновление.

## Принципы

1. Никаких silent destructive-операций.
2. Любая операция сброса выполняется только вручную через Compat/jobs.
3. Все действия совместимости должны быть аудируемыми.
4. Проверка прав выполняется на сервере:
   - `app.compat.view`
   - `app.compat.manage.partial`
   - `app.compat.manage.full`

## Contract для каждого релиза

Каждый релиз обязан содержать:

1. Изменения схемы БД.
2. Новые обязательные ENV.
3. Изменения API/UI совместимости.
4. Требования preflight.
5. Условия отката.

## Критерии отказа обновления

Обновление должно быть остановлено, если:

1. `preflight` содержит `failed` по критичным baseline-check.
2. Бэкап не создан/невалиден.
3. Совместимость вкладок имеет `incompatible` без принятого плана действий.
