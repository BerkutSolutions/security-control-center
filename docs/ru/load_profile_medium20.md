# Минимальный нагрузочный профиль (средняя компания)

Цель: безопасный профиль для локальной машины и пилота без перегруза.

## Профиль Medium-20

- Мониторы: до 20 активных HTTP/TCP.
- Интервал проверки: 30 сек.
- Таймаут: 20-24 сек.
- Повторы: 2.
- Интервал повтора: 30 сек.
- Max concurrent checks: 4.
- Рабочие доски задач: 2-3 доски, 200-500 задач суммарно.
- Инциденты: до 200 открытых/закрытых за 30 дней.

## Рекомендованные настройки

- `BERKUT_MONITORING_STATS_LOG_INTERVAL_SECONDS=60`
- `BERKUT_MONITORING_JITTER_PERCENT=15`
- `BERKUT_MONITORING_JITTER_MAX_SECONDS=10`
- В UI: Monitoring settings -> `max_concurrent_checks = 4`
- Для dev/home отключить тяжелые периодические отчеты и частые bulk-операции.

## Что считать нормой

- CPU в среднем < 35-45% на локальной машине.
- Пики CPU при одновременных таймаутах допустимы до 70%, но кратковременно.
- Потребление RAM стабильное без постоянного роста.

## Быстрый smoke под профиль

1. Создать 20 мониторов с интервалом 30 сек.
2. Проверить 15 минут работу движка.
3. Убедиться, что UI вкладки Monitoring/Tasks/Incidents открываются без лагов.
4. Проверить, что нет лавинообразного роста retry/scheduled checks.

## Ограничения профиля

- Профиль не предназначен для high-cardinality событий (тысячи мониторов).
- Для enterprise-нагрузки нужен отдельный capacity-тест и sizing.
