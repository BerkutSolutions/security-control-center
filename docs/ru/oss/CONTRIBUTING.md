# Как вносить вклад

## Процесс

1. Создайте ветку от актуального mainline.
2. Делайте изменения атомарными и в рамках задачи.
3. Добавляйте/обновляйте тесты при изменении поведения.
4. Обновляйте документацию и `CHANGELOG.md` для user-visible изменений.

## Локальные проверки

Перед merge request запустите:

```bash
go vet ./...
go test -count=1 ./...
```

Если доступен `make`:

```bash
make ci
```

## Инженерные ограничения

- Нельзя добавлять внешние CDN/онлайн-зависимости в рантайме UI.
- Для новых пользовательских строк обязательна локализация RU/EN.
- Для новых endpoint обязательны server-side проверки прав.
- Сохраняйте модульные границы и избегайте избыточно больших файлов.
